Wordpress to Octopress

Feb 26th, 2012

I’ve been using Wordpress for some time now. It’s currently powering my family’s photo blog and a few other sites that I run. I’ve been mostly pleased with it; it’s very handy for creating a static website or blog. However, even with mod_pagespeed, “minify” and caching plugins, it’s not quite as fast as I’d like.

Recently I’ve been hearing lots of good things about “baked” blog solutions. This is where a tool is used to parse your blog content and spit out static html. Presumably this is faster since you’re not performing a kajillion mysql queries every time anyone so much as sneezes at your site. Octopress is a solution that does this and has been gaining lots of traction recently, so I thought I’d give it a try.

My Setup

I’m running Mac OS X 10.7 with Xcode 4.3 (MAS Version), on two personal Macs. I cloned Octopress into my dropbox directory so that I can write from either machine. There were a few “gotchas” I encountered that I will talk about below.

Getting Started

The Octopress docs have very clear and simple instructions on how to get started. You start by getting a particular version of ruby installed and setup. Deployment

I’m deploying to a Dreamhost shared server using the rsync method. I already have SSH keys setup, and it was easy to configure. I’m also using the instructions on that page to host a git repo of this blog in my Dreamhost account.

Beta

Since I have so few existing posts, and I wanted some practice working with Octopress, I migrated each post by hand. If you have a larger blog, you might consider using exitwp.

I setup beta.mattmorr.is to test this out without disturbing my existing Wordpress installation. Once I got it working, I decided I would try to migrate wordpress to legacy.mattmorr.is. I found these complicated instructions. In fact, I was so daunted by this process that I just decided to ‘rm -rf’ the whole site and delete the mysql database.

In contrast, to get Octopress to switch from the beta subdomain to the real one, I just had to make two edits. One in _config.yml, to set the new url, and once in Rakefile, to set the deployment directory on my dreamhost account. Then, after a new redeploy, the site was up and running on the new domain. So much easier than Wordpress.

The Verdict

My blog now feels much lighter and faster (I’m still running mod_pagespeed). I was happy to find that Octopress supports several 3rd party services, including Twitter, Google+, and Google Analytics.

Another unexpected benefit is that it’s really fun to write in Markdown format. I like writing in vi, but I hate editing html. Markdown is simple, intuitive, and easy to learn. I also really like that my writing is now freed from the bloated wordpress database. I can take it to another platform should I wish, and it will be easy to archive my work since it’s not tightly coupled with my web hosting provider. For this reason, I’ll likely convert my family photo blog to Octopress at some point so that it can be more readily archivable for my hypothetical great grandchildren.

TL;DR: I like Octopress. Markdown is cool.
