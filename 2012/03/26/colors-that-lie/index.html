When Rainbows Attack

Mar 26th, 2012

After reading some of the recent commentary regarding Consumer Reports’ sensationalistic report of the New iPad’s running temperature, I would like to also point out the problems that come with using a rainbow color map in a visualization such as this.

A color map is simply a look-up table that maps a range of values into a range of colors. Color maps are frequently used in medical and other scientific visualizations to achieve expressive visualizations. A carefully chosen color map can reveal important characteristics of the data under investigation. But a poor color map can actually introduce artifacts into the final image.

Consumer Reports has chosen the canonical bad color map.

Rainbows, Rainbows Everywhere

They have chosen a rainbow color map to visualize the surface temperature on the New iPad. Loosely speaking, a rainbow color map traverses all the “colors of the rainbow.” For example, given a data set with values between 0 and 255, a rainbow map migh map the colors as follows: 0:blue, 100:green, 200:yellow, 255:red. Then the final image is rendered by taking the data values and drawing color pixels on the screen using the map. The problem with this is that the human eye perceives the transitions between the different colors as discrete events. This image, for example, maps values 0 to 300 using a rainbow map that I created in Pixelmator.

See the transition between red and yellow? Surely there is something significant in the data going on there, right? Right??

No. That “line” between the red and yellow is created in the eye and mind of the viewer and is due to the way that I constructed the color map. Here is the same [0,300] data rendered with a saturation based map.

Imagine if the underlying data were temperature values. This would make it appear that there are sudden temperature changes at certain locations, when in fact the temperature change is very smooth. And it certainly makes a more dramatic image if you map the higher temperatures to the red/orange colors, yes?

If you give me a dataset and enough time to play with the color transform, I can make all kinds of crazy features emerge in the image.

To their credit, Consumer Reports does, at least, appaer to use the same color map for both visualizations. However, it seems very convient that the “cooler” iPad is colored with mostly blue hues, which fits the narrative very nicely.

There is a mountain of visualization research that explains why rainbow color maps are horrible. To get the complete story, I recommend Rainbow Color Maps (Still) Considered Harmful by Borland and Taylor. I think this quote explains the situation pretty well.

Not only does the rainbow color map confuse viewers through its lack of perceptual ordering and obscure data through its inability to present small details, but it actively misleads the viewer by introducing artifacts to the visualization. The rainbow color map appears as if itâ€™s separated into bands of almost constant hue, with sharp transitions between hues. Viewers perceive these sharp transitions as sharp transitions in the data, even when this is not the case.
